## Color Rules (Enforced)

### Strict Palette Compliance

**Use only the provided color palette â€” never create or modify colors.**

- All colors must come from the user-provided palette
- Do not use colors outside the palette
- Do not modify palette colors (brightness, saturation, blending)
- **Only exception**: use the `transparency` property (0-100) to add opacity

```javascript
// âś… Correct: use colors from the palette
slide.addShape(pres.shapes.RECTANGLE, { fill: { color: theme.primary } });
slide.addText("Title", { color: theme.accent });

// âťŚ Wrong: use colors outside the palette
slide.addShape(pres.shapes.RECTANGLE, { fill: { color: "1a1a2e" } });
```

### No Gradients

**Gradients are prohibited â€” use flat colors only.**

### No Animations

**Animations and transitions are prohibited.** All slides must be static.

### Body Text Not Bold

**Body text and caption/legend text must not be bold.**

- Body paragraphs, descriptions â†’ normal weight
- Captions, legends, footnotes â†’ normal weight
- Bold only for headings and subheadings

```javascript

slide.addText("Body content.", { bold: false, fontSize: 14, fontFace: "Arial" });

// âťŚ Wrong
slide.addText("Body content.", { bold: true, fontSize: 14 });
```

## Style Recipe

By adjusting corner radius (`rectRadius`) and spacing, the same design can achieve 4 different visual styles. Choose the recipe that matches your presentation's tone.

> **Units**: PptxGenJS uses inches. Slide size is 10" x 5.625" (LAYOUT_16x9).

### Style Overview

| Style | Corner Radius | Spacing | Best For |
|-------|--------------|---------|----------|
| **Sharp & Compact** | 0 ~ 0.05" | Tight | Data-heavy, tables, professional reports |
| **Soft & Balanced** | 0.08" ~ 0.12" | Moderate | Corporate, business presentations, general use |
| **Rounded & Loose** | 0.15" ~ 0.25" | Loose | Product demos, marketing, creative showcases |
| **Pill & Open** | 0.3" ~ 0.5" | Open | Brand showcases, launches, premium presentations |

### Sharp & Compact

**Visual character**: Strong geometry, high information density, formal and precise.

| Category | Value (inches) | Notes |
|----------|---------------|-------|
| Corner â€” Small | 0" | Sharp corners |
| Corner â€” Medium | 0.03" | Micro-rounded |
| Corner â€” Large | 0.05" | Slightly rounded |
| Element padding | 0.1" ~ 0.15" | Compact |
| Element gap | 0.1" ~ 0.2" | Compact |
| Page margin | 0.3" | Narrow |
| Section gap | 0.25" ~ 0.35" | Compact |

### Soft & Balanced

#### PptxGenJS Corner Radius Examples

```javascript
// Sharp style card
slide.addShape("rect", {
  x: 0.5, y: 1, w: 4, h: 2.5,
  fill: { color: "F5F5F5" },
  rectRadius: 0.03
});

// Rounded style card
slide.addShape("rect", {
  x: 0.5, y: 1, w: 4, h: 2.5,
  fill: { color: "F5F5F5" },
  rectRadius: 0.2
});

// Pill style button (h: 0.4", rectRadius: 0.2" = perfect pill)
slide.addShape("rect", {
  x: 3, y: 4, w: 2, h: 0.4,
  fill: { color: "4A90D9" },
  rectRadius: 0.2
});
```

### Mixing Rules

#### 1. Outer container radius >= inner element radius

```javascript
// âś… Correct: outer > inner
card:   rectRadius: 0.2
button: rectRadius: 0.1

// âťŚ Wrong: inner > outer â†’ visual overflow
card:   rectRadius: 0.1
button: rectRadius: 0.2
```

#### 2. Information density determines spacing

| Area type | Recommended style |
|-----------|------------------|
| Data display area | Sharp / Soft (compact spacing) |
| Content browsing area | Rounded / Pill (loose spacing) |
| Header area | Soft / Rounded (moderate spacing) |

#### 3. Corner radius vs. element height

| Element height | Sharp | Soft | Rounded | Pill |
|---------------|-------|------|---------|------|
| Small (< 0.3") | 0" | 0.03" | 0.08" | height/2 |
| Medium (0.3"~0.6") | 0.02" | 0.05" | 0.12" | height/2 |
| Large (0.6"~1.2") | 0.03" | 0.08" | 0.2" | 0.3" |
| Extra Large (> 1.2") | 0.05" | 0.12" | 0.25" | 0.4" |

> **Pill tip**: Set `rectRadius = element height / 2` for a perfect pill shape
