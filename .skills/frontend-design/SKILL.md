# Frontend Design

This skill guides creation of distinctive, production-grade frontend interfaces that avoid generic "AI slop" aesthetics. Implement real working code with exceptional attention to aesthetic details and creative choices.

## When to Use This Skill

**Trigger conditions:**
- User asks to build: web component, page, application, interface
- User mentions: website, landing page, dashboard, React component, HTML/CSS
- User wants to style, beautify, or improve UI
- User says: "build a UI", "create a component", "make it look good"

## Design Thinking

Before coding, understand the context and commit to a BOLD aesthetic direction:

### Purpose
- What problem does this interface solve?
- Who uses it?
- What's the context (fun/professional, playful/serious)?

### Audience
- Developer peers?
- End users?
- Business stakeholders?

### Aesthetic Direction
Choose a specific mood:
- Minimal and refined?
- Bold and dramatic?
- Playful and experimental?
- Professional and trustworthy?

## Design Principles

### Typography
- Choose distinctive typefaces, not generic ones
- Create hierarchy through type weight, size, and spacing
- Use large display text for impact
- Consider type pairing (one display + one body)

### Color
- Go beyond default palettes
- Create cohesive, intentional schemes
- Use color strategically for emphasis
- Consider light and dark modes

### Backgrounds & Visual Details
- Create atmosphere and depth
- Add contextual effects and textures
- Consider: gradient meshes, noise textures, geometric patterns
- Use layered transparencies, dramatic shadows, custom cursors

### Motion
- Use animations for effects and micro-interactions
- Prioritize CSS-only for HTML
- Use Motion library for React when available
- Focus on high-impact moments with staggered reveals

### Spatial Composition
- Unexpected layouts
- Asymmetry and overlap
- Diagonal flow
- Grid-breaking elements
- Generous negative space OR controlled density

## Anti-Patterns to Avoid

**NEVER use:**
- Generic font families (Inter, Roboto, Arial, system fonts)
- Cliched color schemes (purple gradients on white)
- Predictable layouts and component patterns
- Cookie-cutter design that lacks character

**NEVER converge on common choices** across generations. Vary between:
- Light and dark themes
- Different fonts
- Different aesthetics

## Implementation Notes

### Match Complexity to Vision
- Maximalist designs need elaborate code with extensive animations
- Minimalist designs need restraint, precision, and careful attention to spacing
- Elegance comes from executing the vision well

### Remember
- Claude is capable of extraordinary creative work
- Don't hold back
- Show what can truly be created when thinking outside the box
- Commit fully to a distinctive vision

## Output

Generate complete, working code:
- HTML/CSS for static interfaces
- React components with proper structure
- Responsive and accessible
- Animations and effects where appropriate
